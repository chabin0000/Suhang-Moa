# 선택한 반 재접속 복원 설계

## 목표

사용자가 학년과 반을 한 번 선택하면 같은 브라우저에서 다시 접속하거나 새로고침해도 반 선택 화면을 거치지 않고 해당 반의 캘린더를 바로 표시한다.

## 동작 규칙

1. 앱을 처음 실행할 때 `localStorage`의 `classmap:selectedClass` 값을 읽는다.
2. 저장값이 유효한 학년과 반이면 캘린더 화면을 첫 화면으로 표시한다.
3. 저장값이 없거나 형식이 잘못됐으면 반 선택 화면을 표시한다.
4. 사용자가 반을 선택하면 선택값을 저장하고 해당 반의 캘린더를 표시한다.
5. 캘린더의 `반 변경` 버튼을 눌렀을 때만 반 선택 화면으로 이동한다.
6. 새 반을 선택하면 기존 저장값을 새 선택값으로 교체한다.

## 구현 범위

`App` 컴포넌트의 초기 화면 상태를 저장된 반의 존재 여부에 따라 결정한다. 기존 `getSelectedClass`와 `saveSelectedClass`를 그대로 사용하며, 화면 구성이나 일정 저장 방식은 변경하지 않는다.

## 예외 처리

`getSelectedClass`의 기존 유효성 검사를 사용한다. JSON 파싱에 실패하거나 학년·반 범위를 벗어난 저장값은 없는 값으로 취급해 반 선택 화면을 표시한다.

## 확인 항목

- 처음 접속하면 반 선택 화면이 표시된다.
- 반을 선택하면 해당 반의 캘린더가 표시된다.
- 새로고침하거나 브라우저를 닫았다 다시 접속하면 같은 반의 캘린더가 즉시 표시된다.
- `반 변경` 버튼을 누르면 선택 화면으로 이동한다.
- 다른 반을 선택한 뒤 재접속하면 새로 선택한 반의 캘린더가 표시된다.
- 잘못된 저장값이 있어도 앱이 멈추지 않고 반 선택 화면이 표시된다.

## 제외 범위

브라우저나 기기가 달라졌을 때 선택값을 동기화하는 기능, 로그인 기능, 서버 저장 기능은 이번 변경에 포함하지 않는다.
