# ClassMap MVP

## 실행 링크

- GitHub Pages: https://chabin0000.github.io/-/

## 한 줄 소개

ClassMap은 고등학생이 가입 없이 로컬 브라우저에서 학년/반별 수행평가, 지필평가, 과제, 준비물 일정을 관리하는 캘린더 웹앱 MVP입니다.

## 프로젝트 목적

이 프로젝트는 실제 서비스 출시용이 아니라 학교 수행평가 캘린더 아이디어를 빠르게 검증하기 위한 초안 프로토타입입니다. 서버, DB, 로그인, 회원가입 없이 브라우저의 `LocalStorage`만 사용합니다.

## 사용 기술

- Vite
- React
- TypeScript
- CSS
- Lucide React 아이콘
- Browser LocalStorage

## 설치 방법

```bash
npm install
```

현재 Codex 실행 환경에서는 `node`와 `npm`이 일반 PATH에 없고 `pnpm`만 감지될 수 있습니다. 그 경우 아래 명령을 사용해도 됩니다.

```bash
pnpm install
```

## 실행 방법

```bash
npm run dev
```

또는:

```bash
pnpm dev
```

실행 후 터미널에 표시되는 로컬 주소를 브라우저에서 엽니다. 기본적으로 `http://localhost:5173` 형식입니다.

## 테스트 방법

1. 첫 화면에서 `1학년`, `3반`을 선택하고 `우리 반 들어가기`를 누릅니다.
2. 오른쪽 아래 `일정 추가` 버튼을 눌러 제목, 유형, 마감일을 입력하고 저장합니다.
3. 달력과 일정 목록에 새 일정이 바로 표시되는지 확인합니다.
4. 브라우저를 새로고침한 뒤 일정이 유지되는지 확인합니다.
5. `반 변경`을 눌러 다른 반으로 들어간 뒤 기존 일정이 섞이지 않는지 확인합니다.
6. 요약 위젯 `오늘 마감`, `내일 마감`, `이번 주 일정`을 눌러 목록 필터가 동작하는지 확인합니다.
7. 일정의 `삭제` 버튼을 눌러 삭제 확인창과 LocalStorage 갱신을 확인합니다.

## 현재 구현된 기능

- 학년 선택: 1학년, 2학년, 3학년
- 반 선택: 1반부터 12반
- 선택한 학년/반 LocalStorage 저장
- 재방문 시 이전 학년/반 자동 선택
- 현재 선택한 반의 일정만 조회
- 일정 추가 모달
- 일정 삭제
- 월간 캘린더
- 이전 달/다음 달 이동
- 날짜 클릭 시 해당 날짜가 마감일로 자동 입력
- 오늘 날짜 표시
- 오늘/내일/이번 주 요약 위젯
- 요약 위젯 기반 일정 목록 필터
- 모바일 우선 반응형 화면

## 아직 구현하지 않은 기능

- 서버 DB
- 로그인
- 회원가입
- 실제 반 공유 링크
- 실제 Web Push 알림
- 악성 이용 방지 기능
- Supabase/Firebase 연동
- 관리자 기능
- 신고 기능
- 실시간 동기화
- 학교 검색 기능

## 파일별 역할

- `src/App.tsx`: 선택 화면과 캘린더 화면을 전환하는 최상위 컴포넌트입니다.
- `src/types.ts`: 학년/반, 일정, 일정 유형 타입을 정의합니다.
- `src/utils/storage.ts`: LocalStorage 읽기/쓰기와 저장 데이터 검증을 담당합니다.
- `src/utils/date.ts`: 날짜 문자열, 달력 날짜 배열, 이번 주 계산을 담당합니다.
- `src/components/ClassSelectPage.tsx`: 학년/반 선택 첫 화면입니다.
- `src/components/GradeSelector.tsx`: 학년 버튼 묶음입니다.
- `src/components/ClassNumberSelector.tsx`: 반 선택 버튼 묶음입니다.
- `src/components/ClassDashboard.tsx`: 현재 반 캘린더 화면의 상태와 저장 로직을 관리합니다.
- `src/components/SummaryWidgets.tsx`: 오늘/내일/이번 주 요약 위젯입니다.
- `src/components/CalendarMonth.tsx`: 월간 캘린더 UI입니다.
- `src/components/ScheduleModal.tsx`: 일정 추가 입력 모달입니다.
- `src/components/ScheduleList.tsx`: 일정 목록과 삭제 버튼입니다.
- `src/index.css`: Genesis 디자인 스킬 기준의 색상, 간격, 반응형 스타일입니다.

## 작동 원리

학년/반 선택값은 `classmap:selectedClass` 키로 저장됩니다. 일정 목록은 `classmap:schedules` 키에 배열 형태로 저장됩니다. 일정에는 학년과 반 정보가 함께 들어가므로, 캘린더 화면에서는 현재 선택한 학년/반과 일치하는 일정만 필터링해서 보여줍니다.

## 실제 시스템과 MVP의 차이

이 MVP는 같은 브라우저 안에서만 데이터가 유지됩니다. 다른 기기, 다른 브라우저, 시크릿 모드에서는 같은 데이터를 볼 수 없습니다. 실제 학급용 서비스로 확장하려면 서버 DB, 인증, 공유 링크, 권한 관리, 악성 이용 방지, 알림 시스템이 필요합니다.

## 보고서용 요약문

본 프로젝트는 고등학생의 수행평가와 과제 일정을 학급 단위로 관리하기 위한 웹 기반 캘린더 MVP이다. React와 TypeScript로 화면과 데이터 구조를 명확히 분리했고, 서버 없이 브라우저 LocalStorage를 사용하여 학년/반 선택값과 일정 데이터를 저장했다. 사용자는 학년과 반을 선택한 뒤 월간 캘린더에서 일정을 등록, 조회, 삭제할 수 있으며, 오늘 마감·내일 마감·이번 주 일정 요약을 통해 중요한 일정을 빠르게 확인할 수 있다. 실제 서비스와 달리 데이터 공유나 로그인은 구현하지 않았지만, 로컬 환경에서 핵심 사용 흐름을 검증할 수 있는 프로토타입으로 의미가 있다.
